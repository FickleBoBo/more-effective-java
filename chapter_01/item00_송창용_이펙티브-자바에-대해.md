## 이펙티브 자바에 대해

### 정리

**내용 요약**

저자는 언어 학습의 중요한 요소로 문법, 어휘, 용법을 집었고, 그중에서도 `용법`에 대해 다룰 것을 고했다.

1장에서는 자바에 대한 용법에 대해 본격적으로 학습에 들어가기 전에 몇 가지 용어를 언급한다.

자바에 대한 기초적인 부분에 대한 학습을 했었기 때문에, 모르는 단어를 보게되는 일은 없을 것이라 생각했지만,

나열되는 기술 용어에서 처음보는 언어들이 존재했다.

그중에서도, `메소드 시그니처`는 낯선 단어이면서 `메소드 오버라이드` 개념과 어떠한 관련이 있을 것 같았고,

`패키지-프라이빗`은 접근 수준 단계의 일종이지만, 알고 있는 클래스 접근 단계는 public, default, protected, private 밖에 없었기에

새롭게 추가된 접근 수준 단계인지 파악하기 위해, 단어의 뜻을 조사했다.


### 심화 탐구

**출발점**

자바의 기초적인 부분은 전부 공부했다고 생각했지만, 생소한 단어들이 있었다.

#### `메소드 시그니처`

#### `패키지-프라이빗`

이러한 단어들이 이해가 되지 않았고, 이 단어가 가리키는 개념이 정확히 무엇인지 파악하기 위해 조사를 하였다.

**설명**

1. 메소드 시그니처의 개념

+ 메소드 시그니처란 메소드 오버로딩과 관련된 단어이다.

    + 메소드 오버로딩이란, 이름은 같지만 매개변수가 다른 메소드를 추가로 생성하여, 메소드를 호출할 때, 매개변수를 다르게 하여 각각 다른 메소드를 불러올 수 있게 하는 것을 말한다.

    + 메소드 시그니처란, 복잡하지 않은 개념으로 메소드의 `매개변수 리스트` 일컫는 말이다.

    + 즉, `메소드 오버로딩`이란 이름이 같은 두 메소드의 `메소드 시그니처`를 다르게 설정하는 것이다.


2. 패키지-프라이빗의 개념

+ 패키지-프라이빗이란 클래스에 대한 `접근 수준 단계`의 하나이다.

    + `패키지-프라이빗`을 설명하는 글에서 `아무것도 명시하지 않은 접근 수준` 부분을 보고 해당 단어가 무엇을 가리키는지 유추할 수 있었따.

    + 접근 수준 단계는 private, default, protected, public으로 4가지가 존재한다.

        + 그리고 클래스에 접근 수준을 따로 지정해주지 않으면 default값이 자동으로 설정된다.

        + 책에서 설명하는 `패키지-프라이빗`은 4가지 접근 수준 단계 중 `default` 를 가리키는 것이다.

        + default 접근제어자는 해당 클래스는 `동일한 패키지 안에서만` 접근이 가능하도록 접근을 제어한다.
